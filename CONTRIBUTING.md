# Contributing to cli-interactive

Thank you for your interest in contributing!

## Prerequisites

- Node.js 18+
- npm 9+

## Setup

```sh
git clone https://github.com/tristan-laroubine/cli-interactive.git
cd cli-interactive
npm install
```

## Development workflow

```sh
# Type-check without emitting
npm run typecheck

# Lint (ESLint + typescript-eslint strict rules)
npm run lint
npm run lint:fix

# Run tests (Vitest)
npm test

# Watch mode
npm run test:watch

# Coverage report
npm run test:coverage

# Build (tsc → dist/)
npm run build
```

## Project structure

```
src/
  command.ts       # InteractiveCLI class — core library code
  command.test.ts  # Tests for InteractiveCLI
  index.ts         # Example usage (not exported)
  index.test.ts    # Smoke test for the public export
dist/              # Compiled output (git-ignored, generated by tsc)
```

## Commit convention

This project follows [Conventional Commits](https://www.conventionalcommits.org/).
Commit messages are validated automatically by [commitlint](https://commitlint.js.org/) via a Husky `commit-msg` hook.

```
feat: add support for ZodArray prompts
fix: handle non-interactive TTY gracefully
docs: update contributing guide
test: add edge case for optional boolean fields
chore: bump zod to 4.5
```

Valid types: `feat`, `fix`, `docs`, `test`, `chore`, `refactor`, `perf`, `style`, `ci`, `build`, `revert`.

## Running the example locally

```sh
# With ts-node / tsx / bun
bun run src/index.ts

# Or build first then run
npm run build && node dist/index.js --name Alice --env production
```

## Publishing (maintainers only)

Publishing is guarded by `prepublishOnly`, which runs `typecheck → lint → test → build` before any publish attempt.

### Authenticate

```sh
npm login
```

### Release a new version

```sh
# Bump version (patch | minor | major)
npm version patch

# Publish to the public npm registry
npm publish
```

> The `files` field in `package.json` ensures only the `dist/` folder is shipped to consumers.

### Pre-release (beta / RC)

```sh
npm version prerelease --preid=beta
npm publish --tag beta
```

### Dry run — inspect what will be published

```sh
npm pack --dry-run
```
